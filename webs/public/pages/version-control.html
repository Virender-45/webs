My Git Workflow

Commit Discipline

Good Commit Habits

Atomic commits

One logical change per commit — not "fixed stuff" but "fix: resolve nav z-index overlap on mobile".

Conventional commits

Prefix messages with feat:, fix:, docs:, refactor: for a scannable, semantic log.

.gitignore discipline

Never commit node_modules, .env files, or editor config — keep the repo lean.

README always current

Documentation lives next to code so it never falls out of sync.
