How I Learn

My Learning Process

🔍

Understand First

Before writing a line of code, understand WHY something works — not just how to copy it.

🛠️

Build to Learn

The fastest way to learn any technology is to use it to build something real — even if it's messy at first.

🐛

Debug Patiently

Treating every bug as a puzzle, not a setback — isolate, reproduce, hypothesise, test.

📖

Read Docs

MDN, official docs, and source code before Stack Overflow. Understanding the source beats copying the answer.

🔄

Iteration

Ship a rough version, get feedback (real or self-critique), refine. Repeat until proud.

🤝

Teach Others

The best proof of understanding is being able to explain it simply. Teaching cements knowledge.
